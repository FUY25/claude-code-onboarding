# Lesson 2: Meet Claude

## Goal
The user has their first real conversation with Claude Code and understands how to ask it things naturally.

## Delivery instructions

Say:
> "Lesson 2 is simple: you're going to have a real conversation with Claude. Not a practice one — an actual one.
>
> Claude Code understands plain English. You don't need special commands or 'prompt engineering'. Just ask like you'd ask a smart colleague."

---

## Activity

Say:
> "Try one of these — or make up your own:"
>
> - `What files are in this folder and what are they for?`
> - `I'm new to this. What can you help me do?`
> - `Summarise what this project is about in one sentence.`
>
> "Type one of those (or your own question) and press Enter. I'll respond as Claude Code normally would — because I am Claude Code."

After the user asks their question, answer it naturally and helpfully based on what's actually in the project folder.

Then say:
> "That's how Claude Code works. You ask, it does. No special syntax, no manuals to read.
>
> One thing worth knowing: Claude Code can see all the files in whatever folder you opened it in. So if you open it in a folder with your work documents, it can read and work with those files.
>
> Lesson 2 done. You now know how to have a real conversation with Claude Code.
>
> Ready for Lesson 3? This is where it gets interesting — you're going to make Claude build something for you."

## Error handling

If the user seems unsure what to type:
> "There's no wrong answer here. Literally type: `What is this folder?` and press Enter. I'll handle the rest."

If the user types a command instead of a question (e.g., `ls`):
> "That works too — you can mix commands and questions freely in Claude Code. But try typing a plain-English question now so you can see how that feels."

## Completion

Mark lesson 2 complete in progress.json when the user confirms they're ready for Lesson 3.
